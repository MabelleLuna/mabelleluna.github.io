The Experience App

The Experience

A distributed application that uses camera vision technology to implement a table monitoring system with a Host UI, Server UI, and a Manager UI for sit down restaurants to monitor customer wait times and other necessary data per UI. This app also includes facial recognition for employee and manager login.

Login Page

For the facial recognition program to work, it requires getting the front end to access the camera. I used AJAX to create a media stream asynchronously so the camera could be opened up like a webcam and our program could access it.

The picture below is a prototype of when I first got the camera to run on our login page.

The login page opens the camera and asks for permissions so the camera can be accessed by our openCV program, and the program returns the top 3 IDs it thinks the user may be. We use the library Libpqxx, a C++ client API for Postgres, to connect to our database with C++ on our server. These IDs are grabbed and returned as JSON objects and shown like the picture below.

Host User Interface

The Host user interface is a page meant for hosts/hostesses to be able to add new parties to a wait list and show how long the party is waiting. The timers, party IDs, number of guests, and server assigned are stored in a database where the our camera vision(openCV) program grabs the data. When a host sits the customer down, the openCV program will detect the host going to the table and gets a flag for a timer to start on the server's UI. This is essential for the server that serves that table.

Server User Interface

The Server's user interface includes their assigned tables, along with a timer that changes colors depending on how long they have been waiting. This is meant for showing how long a customer has been waiting to be served. Each time the server visits the table, the camera detects this and resets the timer. Essentially, the server uses this UI to check whether she/he has tables that have been waiting or have not been checked on. This page refreshes as the server logs out and goes to check on her/his table.

Tracking System

Casino Player Tracking System

An application with an RDBMS(Relational Database Management System) used to manage a casino player tracking system. This includes a casino player's information, player history, and table game information. This is meant to be used for casino managers to track and manage employees and for players to use to keep track of their game history. This system uses PostgreSQL for the database, PHP for the back end, and HTML, CSS, and Javascript.

Manager's User Interface

Below is the manager's user interface where the manager can view two sections: Poker and Table games. Each game shows the amount of hours played in the month. The bottom left shows top players at the casino.

Manager's UI: Game Page

This page shows the hours played by game limit for each game.

On the same page as you scroll down it includes the top 10 most played games. Below that is the Average Session Length which shows that average session lengths for players.

Manager's UI: Player Data Page

This page shows the manager all player's data and gives the manager the ability to Add a new player or Delete a player in the database.

When clicking the the Add or Delete buttons on this page, a modal form will pop up.

For the Add Player form, the manager can add a new player to the database by entering in the player's info.

For the Delete Player form, the manager can also delete a player from the database.

Manager's UI: Employee Data Page

Similar to the player page, this page gives all employee's data and gives the manager the ability to add or delete an employee from the database and also gives a similar modal form for both.

Player's User Interface

The player's UI will allow the player to view their information, along with games played and their play time.
